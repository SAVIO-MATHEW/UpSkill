## Why `.js` still works with JSX

Modern setups (CRA, Vite, Next.js, Webpack) configure Babel to:

* parse JSX in **both `.js` and `.jsx`**

So this is valid:

```js

  return <h1>Hello</h1>;
}
```

### Interview-ready answer üß†
