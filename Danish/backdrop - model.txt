## What is a backdrop?

A **backdrop** is the semi-transparent layer **behind the modal** that:

* blocks interaction with the page
* visually focuses the user on the modal
* gives a **clickable area to close the modal**

## Why you need it (UX + logic)

### Without backdrop ‚ùå

* User can click the page behind the modal
* Outside-click detection becomes unreliable
* Modal feels ‚Äúfloating‚Äù and broken

### With backdrop ‚úÖ

* Clear focus
* Safe outside-click area
* Standard behavior users expect (Netflix, Gmail, etc.)

## How backdrop works in code

```jsx
<div className="backdrop" onClick={onClose}>
  <div
    className="modal"
    onClick={(e) => e.stopPropagation()}
  >
    Modal content
  </div>
</div>
```

### What‚Äôs happening

* Clicking **backdrop** ‚Üí closes modal
* Clicking **modal** ‚Üí `stopPropagation()` prevents closing

üëâ This often **replaces document event listeners** entirely.

  inset: 0;

## When to use which?

| Approach           | Use when                   |
| ------------------ | -------------------------- |
| Backdrop click     | Most apps (recommended)    |
| Document listeners | Portals / complex overlays |
| Both               | Rare, advanced cases       |
