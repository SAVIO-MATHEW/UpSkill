## üß© How JavaScript Works Step by Step

### 1. **Execution Environment**
- JavaScript runs inside a **runtime environment** such as a web browser or Node.js.
- Browsers provide APIs (like `document`, `window`, `fetch`) that JavaScript can use to interact with the page and the internet.

### 2. **JavaScript Engine**
- Each browser has its own engine:
  - **V8** (Chrome, Edge, Node.js)
  - **SpiderMonkey** (Firefox)
  - **JavaScriptCore** (Safari)
- The engine parses JavaScript code, compiles it into machine code, and executes it.

### 3. **Compilation & Execution**
- JavaScript is often described as an *interpreted* language, but modern engines use **Just-In-Time (JIT) compilation**:
  - Code is parsed into an Abstract Syntax Tree (AST).
  - Optimizations are applied.
  - Machine code is generated and executed quickly.

### 4. **Event Loop & Concurrency**
- JavaScript is **single-threaded** but uses an **event loop** to handle asynchronous tasks.
- Tasks like network requests, timers, or user input are queued and executed when the main thread is free.
- This allows JavaScript to remain responsive without blocking.

### 5. **DOM Manipulation**
- JavaScript can directly change the structure and style of a webpage by interacting with the **Document Object Model (DOM)**.
- Example:  
  ```javascript
  document.getElementById("title").innerText = "Hello World!";
  ```
  This updates the text of an element instantly.

### 6. **Client-Side & Server-Side**
- **Client-side:** Runs in the browser to create interactivity (animations, forms, maps).
- **Server-side:** With Node.js, JavaScript can handle backend logic, databases, and APIs.

## üö® Challenges & Trade-offs
- **Single-threaded nature**: Heavy computations can block the UI.
- **Browser differences**: Engines may behave slightly differently.
- **Security risks**: Running arbitrary scripts can expose vulnerabilities if not handled properly.
