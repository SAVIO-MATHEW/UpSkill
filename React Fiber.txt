# üî• What is React Fiber?

React Fiber is the **new reconciliation engine** introduced in React 16.
It is a complete rewrite of React‚Äôs core algorithm that makes rendering **interruptible, incremental, and prioritized**.

---

# üß† Why Fiber Was Introduced

Before Fiber:

* Rendering was synchronous.
* Once rendering started, it could not be paused.
* Large component trees blocked the main thread.
* UI could freeze.

Fiber solves this by:

* Breaking rendering work into small units.
* Allowing React to pause, resume, and prioritize updates.

---

# üî• What is a ‚ÄúFiber‚Äù?

A Fiber is a **JavaScript object representing a unit of work**.

Each React component corresponds to a Fiber node.

Fiber stores:

* Component type
* Props
* State
* Parent/child/sibling references
* Effect tags (what needs to change)

It forms a linked list tree structure.

---

# üî• How React Fiber Works (High Level)

React divides work into two phases:

## 1Ô∏è‚É£ Render Phase (Reconciliation)

* Builds a new Fiber tree.
* Can be paused.
* Can be aborted.
* Determines what changes are needed.

## 2Ô∏è‚É£ Commit Phase

* Applies changes to the DOM.
* Cannot be interrupted.
* Runs lifecycle methods and effects.

---

# üî• Why Fiber Improves Performance

Because it:

* Splits work into small chunks.
* Uses requestIdleCallback (conceptually).
* Assigns priority to updates.
* Allows high-priority tasks (like typing) to interrupt low-priority tasks.

This enables:

* Concurrent Rendering
* Suspense
* Time slicing
* Better UX

# üî• Virtual DOM (VDOM)

Virtual DOM is a **concept**.

It is a lightweight JavaScript representation of the real DOM.

When state changes:

1. React creates a new Virtual DOM tree.
2. Compares it with the previous one (diffing).
3. Updates only the changed parts in the real DOM.

üëâ Goal: Minimize expensive DOM operations.

---

# üî• React Fiber

Fiber is the **engine that powers reconciliation**.

It is the internal algorithm and data structure React uses to:

* Build the Virtual DOM tree
* Compare trees
* Schedule updates
* Prioritize work
* Pause & resume rendering

üëâ Fiber controls *how and when* Virtual DOM updates happen.

---

# üß† Simple Analogy

* Virtual DOM = Blueprint of UI
* Fiber = Construction manager deciding:

  * What to build first
  * When to pause
  * What has higher priority

---

# üî• Key Difference

| Virtual DOM             | React Fiber                          |
| ----------------------- | ------------------------------------ |
| Concept                 | Implementation engine                |
| Represents UI tree      | Represents unit of work              |
| Used for diffing        | Used for scheduling + reconciliation |
| Existed before React 16 | Introduced in React 16               |

---

# üî• Very Important Clarification

Virtual DOM existed **before Fiber**.

Old React (Stack Reconciler):

* Used Virtual DOM
* But rendering was synchronous

Fiber:

* Still uses Virtual DOM
* But adds scheduling + prioritization
