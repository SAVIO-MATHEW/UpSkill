## Short answer

> A function component can have **any number of `useEffect` hooks** because each `useEffect` is an **independent side-effect subscription**, not a lifecycle replacement.

---

## How this works

Think of a function component as:

* a **description of UI**
* plus a list of **effects to run after commit**

Each `useEffect` call:

* registers **one effect**
* with its own dependencies
* and its own cleanup

```jsx
function Component() {
  useEffect(() => {
    // effect A
    return () => cleanupA();
  }, [a]);

  useEffect(() => {
    // effect B
    return () => cleanupB();
  }, [b]);

  return <UI />;
}
```

React tracks them **by call order**, not by name.
